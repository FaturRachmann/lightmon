## Mengapa lightmon?

Kebanyakan system monitor modern seperti `htop`, `btop`, dan `glances` sangat powerful ‚ÄĒ tapi juga berat, lambat startup, dan penuh fitur yang jarang dipakai. **lightmon** mengambil pendekatan berbeda: ambil hanya yang penting, eksekusi dengan sempurna, dan jaga agar binary tetap kecil.

> *"Fast by default. Advanced by design."*

## Fitur

### ūüĖ•ÔłŹ CPU Monitor
- Usage total sistem secara real-time
- **Toggle per-core** ‚ÄĒ lihat beban tiap core secara individual (tekan `e`)
- Deteksi jumlah core otomatis via `gopsutil`
- Progress bar dengan warna adaptif berdasarkan threshold

### ūüß† Memory Monitor
- RAM: used / total dengan persentase
- **Swap memory** ‚ÄĒ monitoring terpisah, muncul otomatis jika swap aktif
- Format bytes cerdas: otomatis tampil dalam KB / MB / GB sesuai ukuran

### ūüíĺ Disk Monitor
- Deteksi **semua partisi** yang ter-mount secara otomatis
- Deduplikasi per device (tidak ada duplikat entry untuk partisi yang sama)
- Tampil path mount point, used, free, dan persentase pemakaian
- Skip partisi virtual (total = 0)

### ūüĆź Network Monitor
- **TX / RX rate** per detik (bytes/sec) ‚ÄĒ dihitung dari delta antar polling
- Akumulasi total bytes sent dan received sejak boot
- Support multiple interface (eth0, wlan0, dst.)
- Loopback (`lo`) di-skip otomatis

### ‚öôÔłŹ Process Manager
- Tampilkan top N proses (default 20, configurable via `--procs`)
- **4 mode sorting** ‚ÄĒ langsung ganti tanpa restart:
  - `c` ‚Üí sort by CPU usage
  - `m` ‚Üí sort by Memory usage
  - `p` ‚Üí sort by PID
  - `n` ‚Üí sort by Name (alphabetical)
- Status proses color-coded: Running (hijau), Sleep (cyan), Zombie (merah)
- **Kill process** ‚ÄĒ pilih baris, tekan `k`, konfirmasi ‚Üí kirim SIGTERM

### ūüé® TUI & UX
- Layout 4-panel atas + process table + status bar
- Navigasi keyboard penuh ‚ÄĒ tidak perlu mouse
- Refresh rate configurable (`--interval 500ms` sampai beberapa detik)
- Non-blocking render ‚ÄĒ UI tidak freeze saat pengumpulan data
- Padding, alignment, dan truncation rapi di semua ukuran terminal

## Penggunaan

```
lightmon [flags]
```

| Flag | Default | Deskripsi |
|------|---------|-----------|
| `--interval` | `1s` | Interval refresh. Contoh: `500ms`, `2s` |
| `--procs` | `20` | Jumlah maksimal proses yang ditampilkan |
| `--version` | ‚ÄĒ | Tampilkan versi dan keluar |

**Contoh:**

```bash
lightmon                          # Default ‚ÄĒ 1 detik, 20 proses
lightmon --interval 500ms         # Refresh lebih cepat
lightmon --procs 50               # Tampilkan top 50 proses
lightmon --interval 2s --procs 10 # Refresh lambat, proses sedikit
lightmon --version                # lightmon v1.0.0
```

- **metrics/** ‚ÄĒ stateless, pure functions, tidak ada side effect UI
- **Updater** ‚ÄĒ satu goroutine, polling dengan `time.Ticker`, push ke buffered channel (size 1)
- **Dashboard** ‚ÄĒ consume channel via `app.QueueUpdateDraw()`, thread-safe dengan tview

### Design Decisions

**Mengapa buffered channel size 1?**
Jika render lebih lambat dari polling (terminal lambat), snapshot lama dibuang dan diganti snapshot terbaru ‚ÄĒ tidak ada backpressure, tidak ada memory leak.

**Mengapa `gopsutil`?**
Single dependency untuk semua OS. Abstraksi `/proc/` (Linux) dan `sysctl` (macOS) secara transparan ‚ÄĒ kode metrics tidak perlu tahu platform.

**Mengapa `tview`?**
Built on `tcell`, support keyboard navigation, table selection, dan modal out-of-the-box. Lebih high-level dari raw ANSI escape codes tapi jauh lebih ringan dari Bubble Tea ecosystem.
